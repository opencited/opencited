# Impeccable Design Context

## Design Context

### Users
Solo developers and indie hackers who want to track, analyze, and improve their website's visibility in AI answer engines (ChatGPT, Perplexity, Claude, Gemini). They are technically capable, value transparency and control, and use this tool as a workspace for ongoing optimization—not a passive dashboard they check once a month. Context: desk work during the day, possibly evening sessions; they need to move fast and find insights without friction.

### Brand Personality
Calm, focused, and authoritative. The interface should feel like a well-organized workspace that reduces cognitive load rather than adding to it. Not flashy or attention-grabbing—confident through restraint. Technical but not intimidating.

**Voice**: Precise, minimal, no fluff. Every element earns its place.

**3-word personality**: Calm · Technical · Refined

### Aesthetic Direction
Refined minimal inspired by Linear and Raycast. Light mode as default with dark mode available via toggle. Generous whitespace, subtle shadows, restrained color palette. Typography-forward with a distinctive display font paired with a clean body font. Motion is present but subtle—purposeful entrance animations, no gratuitous effects.

**References**: Linear (calm authority, information density done right), Raycast (polished minimal, keyboard-first feel), Stripe Dashboard (warm neutrals, typographic hierarchy)

**Anti-references**: Busy dashboards with charts everywhere, generic blue/gradient AI tool aesthetics, enterprise-y data visualizations

### Design Principles
1. **Calm density**: Present rich data without overwhelming. Progressive disclosure over wall-of-metrics.
2. **Keyboard-first thinking**: Keyboard shortcuts, efficient navigation, power-user friendly.
3. **Restrained palette**: Zinc/neutral base, single accent color used sparingly. No decorative gradients.
4. **Typography hierarchy**: Distinctive display font for headings, refined body font for readability. Fluid type scale on marketing pages, fixed scale in app UI.
5. **Motion with purpose**: Staggered entrances on page load, subtle state transitions. Nothing bouncy or elastic.
6. **Component reuse**: Always use UI package components (Button, Card, etc.) instead of raw HTML elements. This ensures consistency and allows centralized style changes.
